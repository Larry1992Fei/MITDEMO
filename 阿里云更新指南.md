# 阿里云项目更新部署指南

## 🎯 更新方式选择

### 方案一：Git拉取更新（推荐）

**适用条件**: 阿里云服务器上已经通过git clone部署了项目

**步骤**:
```bash
# 1. SSH连接到阿里云服务器
ssh username@your-server-ip

# 2. 进入项目目录
cd /path/to/your/MITDEMO

# 3. 拉取最新代码
git pull origin main

# 4. 重启Web服务（如果需要）
# 如果使用nginx
sudo systemctl reload nginx

# 如果使用Apache
sudo systemctl reload apache2

# 如果使用Node.js服务
pm2 restart your-app-name
```

### 方案二：文件上传更新

**适用条件**: 服务器上没有Git，或者只想更新特定文件

**步骤**:
```bash
# 使用SCP上传文件
scp -r prototype_new_ux/ username@your-server-ip:/path/to/web/root/

# 或者使用rsync同步（推荐）
rsync -avz --delete prototype_new_ux/ username@your-server-ip:/path/to/web/root/prototype_new_ux/
```

### 方案三：压缩包上传

**适用条件**: 网络较慢，或者需要完整替换

**本地操作**:
```bash
# 1. 创建压缩包（排除不需要的文件）
tar -czf mitdemo-update.tar.gz prototype_new_ux/ --exclude=node_modules --exclude=.git

# 2. 上传到服务器
scp mitdemo-update.tar.gz username@your-server-ip:/tmp/
```

**服务器操作**:
```bash
# 1. SSH连接服务器
ssh username@your-server-ip

# 2. 备份现有项目
cp -r /path/to/web/root/prototype_new_ux /path/to/backup/prototype_new_ux_backup_$(date +%Y%m%d)

# 3. 解压新文件
cd /path/to/web/root/
tar -xzf /tmp/mitdemo-update.tar.gz

# 4. 设置正确的权限
chown -R www-data:www-data prototype_new_ux/
chmod -R 755 prototype_new_ux/
```

## 🔧 常见部署环境配置

### Nginx配置示例
```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/web/root;
    index index.html;

    location /prototype_new_ux/ {
        try_files $uri $uri/ =404;
    }

    location ~ \.(css|js|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### Apache配置示例
```apache
<VirtualHost *:80>
    ServerName your-domain.com
    DocumentRoot /path/to/web/root
    
    <Directory "/path/to/web/root/prototype_new_ux">
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>
```

## 🚨 故障排除

### 如果更新后页面无法访问
```bash
# 检查文件权限
ls -la /path/to/web/root/prototype_new_ux/

# 检查Web服务状态
sudo systemctl status nginx
# 或
sudo systemctl status apache2

# 查看错误日志
sudo tail -f /var/log/nginx/error.log
# 或
sudo tail -f /var/log/apache2/error.log
```

### 如果JavaScript功能异常
1. 打开浏览器开发者工具
2. 检查Console标签页的错误信息
3. 检查Network标签页的资源加载情况
4. 确认js/app-bundle.js文件是否正确加载
