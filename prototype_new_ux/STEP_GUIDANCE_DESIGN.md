# 步骤引导设计方案

## 设计目标
1. **清晰的参数传递流程** - 突出显示每一步产生的关键参数如何传递到下一步
2. **视觉化引导** - 通过动画、高亮、连接线等方式展示步骤间的关联
3. **参数追踪** - 让用户清楚地看到参数的来源和去向
4. **交互式体验** - 提供更好的用户体验和理解

## 核心设计理念

### 1. 参数流转可视化
- **参数卡片**: 每个步骤产生的关键参数以卡片形式展示
- **流转动画**: 参数从一个步骤"流向"下一个步骤的动画效果
- **参数高亮**: 在代码块中高亮显示来自上一步的参数

### 2. 步骤间连接器
- **连接线**: 步骤之间的视觉连接线，显示数据流向
- **参数标签**: 在连接线上标注传递的关键参数名称
- **状态指示**: 显示参数是否已准备就绪

## 详细设计方案

### A. PayerMax托管 + 前置组件模式 (5步流程)

#### Step 1 → Step 2: 订阅参数传递
**传递的参数:**
- `totalPeriods` (总期数)
- `periodUnit` + `periodCount` (扣款周期)
- `amount` + `currency` (金额币种)
- `trialConfig` (试用配置，如适用)
- `trialPeriodConfig` (优惠期配置，如适用)

**引导设计:**
```
┌─────────────────┐    参数卡片    ┌─────────────────┐
│   Step 1        │ ──────────────→ │   Step 2        │
│ 配置订阅参数     │  订阅计划参数   │ 创建订阅计划     │
└─────────────────┘                └─────────────────┘
```

**视觉元素:**
- 参数卡片显示: "订阅计划参数已配置 ✓"
- 流转动画: 参数从Step 1滑动到Step 2
- 代码高亮: Step 2中使用Step 1参数的地方高亮显示

#### Step 2 → Step 3: 订阅单号传递
**传递的参数:**
- `subscriptionNo` (订阅单号)

**引导设计:**
```
┌─────────────────┐   订阅单号     ┌─────────────────┐
│   Step 2        │ ──────────────→ │   Step 3        │
│ 创建订阅计划     │ subscriptionNo │ 加载前置组件     │
└─────────────────┘                └─────────────────┘
```

#### Step 3 → Step 4: Token传递
**传递的参数:**
- `paymentToken` (支付令牌)
- `sessionKey` (会话密钥)

**引导设计:**
```
┌─────────────────┐  Token & Key   ┌─────────────────┐
│   Step 3        │ ──────────────→ │   Step 4        │
│ 加载前置组件     │ paymentToken   │ 后端支付绑定     │
│                 │ sessionKey     │                 │
└─────────────────┘                └─────────────────┘
```

#### Step 4 → Step 5: 激活结果传递
**传递的参数:**
- `paymentTokenID` (支付令牌ID)
- 订阅激活状态

**引导设计:**
```
┌─────────────────┐   激活结果     ┌─────────────────┐
│   Step 4        │ ──────────────→ │   Step 5        │
│ 后端支付绑定     │ paymentTokenID │ 完成订阅激活     │
└─────────────────┘                └─────────────────┘
```

### B. 商户自管 + 前置组件模式 (4步流程)

#### Step 1 → Step 2: 绑定参数传递
**传递的参数:**
- `amount` + `currency` (绑定金额)
- `subject` (订单标题)
- `userId` (用户ID)

#### Step 2 → Step 3: Token传递
**传递的参数:**
- `paymentToken`
- `sessionKey`

#### Step 3 → Step 4: 绑定结果传递
**传递的参数:**
- `paymentTokenID`

## 具体实现方案

### 1. 参数卡片组件
```html
<div class="param-card">
  <div class="param-card-header">
    <span class="param-icon">📋</span>
    <span class="param-title">订阅计划参数</span>
    <span class="param-status ready">✓</span>
  </div>
  <div class="param-card-body">
    <div class="param-item">
      <span class="param-key">totalPeriods:</span>
      <span class="param-value">12</span>
    </div>
    <div class="param-item">
      <span class="param-key">amount:</span>
      <span class="param-value">9.99 USD</span>
    </div>
  </div>
</div>
```

### 2. 步骤连接器
```html
<div class="step-connector">
  <div class="connector-line"></div>
  <div class="connector-label">
    <span class="connector-icon">→</span>
    <span class="connector-text">subscriptionNo</span>
  </div>
</div>
```

### 3. 代码高亮增强
- 在代码块中，来自上一步的参数用特殊颜色高亮
- 添加参数来源提示tooltip
- 参数值实时更新并闪烁提示

### 4. 动画效果
- **参数流转动画**: 参数从一个步骤"飞"到下一个步骤
- **高亮脉冲**: 新传递的参数在目标步骤中脉冲高亮
- **连接线动画**: 连接线上的数据流动效果

## 交互设计

### 1. 参数追踪模式
- 点击任意参数，高亮显示其在所有步骤中的使用位置
- 显示参数的完整传递路径

### 2. 步骤预览
- 在当前步骤显示"下一步将使用的参数"预览
- 显示"来自上一步的参数"回顾

### 3. 参数验证提示
- 实时验证参数的有效性
- 显示参数传递的状态（准备中/已就绪/传递中/已接收）

## 布局设计

### 1. 顶部参数流程条
```
[Step 1] ──→ [Step 2] ──→ [Step 3] ──→ [Step 4] ──→ [Step 5]
   ↓           ↓           ↓           ↓           ↓
订阅参数    subscriptionNo  paymentToken  激活结果    完成
```

### 2. 侧边参数面板
- 显示当前步骤的输入参数
- 显示当前步骤的输出参数
- 参数来源和去向的可视化

### 3. 步骤间过渡页面
- 在点击"下一步"后，显示参数传递的动画
- 展示即将在下一步使用的参数
- 提供"继续"按钮进入下一步

## 技术实现要点

### 1. 状态管理
- 维护全局参数状态
- 追踪参数的传递历史
- 实时更新参数值

### 2. 动画库
- 使用CSS动画或JavaScript动画库
- 实现流畅的参数传递动画
- 响应式动画适配不同屏幕尺寸

### 3. 组件化设计
- 参数卡片组件
- 连接器组件
- 高亮组件
- 动画组件

## 用户体验优化

### 1. 渐进式引导
- 首次访问时显示引导提示
- 可选择开启/关闭参数追踪模式

### 2. 个性化设置
- 允许用户选择动画速度
- 允许用户选择高亮样式
- 保存用户偏好设置

### 3. 错误处理
- 参数缺失时的友好提示
- 参数格式错误时的修正建议
- 传递失败时的重试机制

## 实施优先级

### Phase 1: 基础功能
1. 参数卡片显示
2. 基础连接线
3. 简单的参数高亮

### Phase 2: 增强体验
1. 参数流转动画
2. 交互式参数追踪
3. 步骤间过渡效果

### Phase 3: 高级功能
1. 个性化设置
2. 高级动画效果
3. 智能参数建议

这个设计方案将大大提升用户对订阅代扣流程的理解，让复杂的API调用流程变得直观易懂。
