# 阿里云部署问题诊断 - Step步骤栏不展示

## 🔍 问题分析

**网站地址**: http://mitdemo.jimas.cn/
**问题现象**: 业务流程step步骤栏不展示

### 可能原因分析

1. **JavaScript文件加载失败**
   - `js/app-bundle.js` 文件路径错误或文件缺失
   - 文件权限问题
   - 网络加载超时

2. **CSS样式文件问题**
   - CSS文件加载失败导致步骤栏不可见
   - 样式冲突或覆盖

3. **JavaScript执行错误**
   - 代码语法错误
   - 依赖库加载失败
   - 浏览器兼容性问题

4. **文件路径问题**
   - 相对路径在服务器环境下解析错误
   - 大小写敏感问题（Linux服务器）

## 🚨 常见问题及解决方案

### 问题1: JavaScript文件404错误

**现象**: Network标签显示 `js/app-bundle.js` 404错误

**解决方案**:
```bash
# 检查文件是否存在
ls -la /path/to/web/root/prototype_new_ux/js/app-bundle.js

# 如果文件不存在，重新上传
scp prototype_new_ux/js/app-bundle.js username@server-ip:/path/to/web/root/prototype_new_ux/js/
```

### 问题2: CSS文件加载失败

**现象**: 页面样式混乱，步骤栏不可见

**解决方案**:
```bash
# 检查CSS文件
ls -la /path/to/web/root/prototype_new_ux/css/

# 重新上传CSS文件
scp -r prototype_new_ux/css/ username@server-ip:/path/to/web/root/prototype_new_ux/
```

### 问题3: 外部CDN加载失败

**现象**: PayerMax CDN文件加载失败

**解决方案**: 在index-full.html中添加备用方案或本地文件

### 问题4: 路径大小写问题

**现象**: Linux服务器对大小写敏感

**解决方案**:
```bash
# 检查文件名大小写
find /path/to/web/root/prototype_new_ux/ -name "*.js" -o -name "*.css" -o -name "*.html"
```

## 🔧 快速修复脚本

创建一个快速诊断脚本：

```bash
#!/bin/bash
# 保存为 diagnose.sh

echo "=== 文件结构检查 ==="
ls -la /path/to/web/root/prototype_new_ux/

echo "=== 关键文件检查 ==="
files=(
    "/path/to/web/root/prototype_new_ux/index-full.html"
    "/path/to/web/root/prototype_new_ux/js/app-bundle.js"
    "/path/to/web/root/prototype_new_ux/css/layout.css"
    "/path/to/web/root/prototype_new_ux/css/components.css"
    "/path/to/web/root/prototype_new_ux/css/variables.css"
    "/path/to/web/root/prototype_new_ux/css/code-display.css"
)

for file in "${files[@]}"; do
    if [ -f "$file" ]; then
        echo "✅ $file 存在"
        ls -la "$file"
    else
        echo "❌ $file 缺失"
    fi
done

echo "=== 权限修复 ==="
sudo chown -R www-data:www-data /path/to/web/root/prototype_new_ux/
sudo chmod -R 755 /path/to/web/root/prototype_new_ux/
sudo chmod 644 /path/to/web/root/prototype_new_ux/*.html
sudo chmod 644 /path/to/web/root/prototype_new_ux/js/*.js
sudo chmod 644 /path/to/web/root/prototype_new_ux/css/*.css

echo "=== Web服务器状态 ==="
sudo systemctl status nginx
```
